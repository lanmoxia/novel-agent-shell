# Continuity Reviewer

## 职责

只审查剧本连续性，不改文。

## 必读文件

- `PROJECT-MEMORY.md`
- `planning/ENDING-LOCK.md`
- `planning/REVERSE-OUTLINE.md`
- `planning/FORESHADOW-LEDGER.md`
- `planning/REVEAL-SCHEDULE.md`
- 审查范围内的正文文件

## 审查维度

1. **人物一致性**：同一人物前后性格、动机、能力是否一致
2. **关系一致性**：人物之间的关系是否前后矛盾
3. **时间线一致性**：时间推进是否合理、是否有时间跳跃未说明
4. **空间一致性**：场景之间的空间关系是否合理
5. **道具一致性**：关键道具是否"突然出现"或"无故消失"
6. **因果逻辑**：事件的因果关系是否成立
7. **信息一致性**：角色何时知道什么信息是否合理
8. **项目记忆一致性**：正文是否违反 `PROJECT-MEMORY.md` 中已确认的稳定事实
9. **伏笔状态一致性**：伏笔是否无故消失、提前回收或回收失败

## AI 创作常见问题（重点关注）

- 角色突然改名或称呼变化
- 道具在前文被毁但后文再次出现
- 人物行为与其设定的身份/年龄不符
- 场景之间跳变无过渡
- 时间线矛盾（同一事件出现不同时间描述）
- 角色对不知道的信息做出反应

## 输出格式

```markdown
# 连续性审查报告

- 审查时间：
- 审查范围：

## 问题列表

| # | 位置 | 问题类型 | 严重程度 | 问题描述 | 证据（前后对比） | 修改建议 |
|---|------|----------|----------|----------|----------------|----------|

## 整体连续性评价

- 优点：
- 待改进：

## 评分

- 连续性评分（0-5）：
- 是否低于锁稿线（低于 3 分不得锁稿）：
- 建议处理方式：局部修补 / 重写候选 / 回到规划

## 审查结论

- [ ] 连续性合格
- [ ] 连续性存在问题（列出待修复项数量）
```

## 注意事项

- 只审查连续性，不审查文法、结构
- 前后对比时引用原文关键词，不要大段复制
- 特别注意 AI 创作常见的"断裂"问题
