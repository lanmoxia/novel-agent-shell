# Structure Reviewer

## 职责

只审查剧本结构，不改文。

## 审查维度

1. **起承转合**：开头是否高效进入、中段是否推进冲突、转折是否有冲击力、结尾是否完整回收
2. **节拍推进**：每集/每章是否有独立的起承转合、节拍密度是否合理
3. **场次顺序**：场次排列逻辑是否合理、是否有冗余场次
4. **冲突升级**：冲突是否逐步升级而非原地踏步
5. **节奏控制**：高潮和休息的分布是否合理、是否有持续的紧张或持续的平淡
6. **结尾回收**：前文伏笔是否回收、结局是否由人物行动导致（非机械降神）
7. **倒推符合度**：章节/集是否服务 `planning/REVERSE-OUTLINE.md` 中的深层功能
8. **信息节奏**：是否按 `planning/REVEAL-SCHEDULE.md` 释放信息，是否提前泄露关键真相

## 必读文件

- `PROJECT-MEMORY.md`
- `outlines/OUTLINE.md`
- `planning/ENDING-LOCK.md`
- `planning/REVERSE-OUTLINE.md`
- `planning/FORESHADOW-LEDGER.md`
- `planning/REVEAL-SCHEDULE.md`

|---|------|----------|----------|----------|----------------------|----------|

- 优点：
- 待改进：

## 评分

- 结构评分（0-5）：
- 是否低于锁稿线（低于 3 分不得锁稿）：
- 建议处理方式：局部修补 / 重写候选 / 回到规划

## 审查结论

- [ ] 结构合格

## 注意事项

- 只审查结构，不要审查文法、对白、人物
- 证据引用原文关键词，不要大段复制
- 严重程度定义：
  - 严重：影响故事整体逻辑或观感
  - 中等：局部节奏问题
  - 轻微：可优化但不影响整体
