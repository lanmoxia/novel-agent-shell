# script-review

## 职责

组织和汇总四类审查，生成最终审查报告。

## 前置条件

- `projects/<project-name>/scripts/` 中有已写好的正文
- `projects/<project-name>/GENRE-SKILL-LOCK.md` 存在

## 工作流程

1. 确定审查范围（全部 / 某几集 / 某几章）
2. 依次启动四类审查 Agent：
   - **structure-reviewer**：审查结构
   - **continuity-reviewer**：审查连续性
   - **dialogue-reviewer**：审查对白
   - **genre-reviewer**：审查题材符合度
3. 等待所有审查 Agent 完成
4. 汇总各审查报告
5. 按严重程度排序问题列表
6. 为每个问题标注建议处理方式：局部修补 / 重写候选 / 回到规划
7. 按结构、连续性、对白、题材符合度给出 0-5 分
8. 更新 PROJECT-MEMORY.md 的审查与风险记录
9. 生成 `reviews/final-review.md`

## 输出

```
projects/<project-name>/reviews/structure-review.md
projects/<project-name>/reviews/continuity-review.md
projects/<project-name>/reviews/dialogue-review.md
projects/<project-name>/reviews/genre-review.md
projects/<project-name>/reviews/final-review.md
```

## Phase 5 → Phase 6 门禁规则

审查完成后，如果结论为"有条件通过"或"不通过"，必须遵守以下规则：

1. **更新 SCRIPT-STATUS.md**：将当前状态写为 `审查完成，待用户确认进入 Phase 6`
2. **不得自动调用 revision-fixer**：禁止在用户未明确确认的情况下自动进入 Phase 6 修订
3. **不得自动修改正文**：审查报告只记录问题，不直接修改剧本文件
4. **必须等待用户确认**：用户必须明确选择修订范围后，才允许进入 Phase 6
5. **提供修订选项供用户选择**（不得代替用户决策）：
   - A：仅修复严重问题
   - B：修复严重 + 中等问题
   - C：全部修复（严重 + 中等 + 轻微）

## 注意事项

- 审查 Agent 只读自己负责范围内的必要文件
- 审查必须读取 PROJECT-MEMORY.md，连续性审查尤其必须以项目记忆为基准
- 任一维度低于 3 分不得建议锁稿
- 中等或严重问题必须给出是否建议重写候选
- 审查报告不要重复输出原始内容
- 严重问题的定义：影响故事逻辑、人物一致性、题材合规
