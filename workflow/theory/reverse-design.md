# 倒推式故事设计

## 核心原则

正式项目应先确定终点，再生成过程。短篇、长篇、剧本和短剧都适用，只是粒度不同。

倒推起点包括：
- 最终结局事实
- 核心真相
- 情绪落点
- 主角最终面对的问题
- 不可提前暴露的信息

## 使用阶段

- Phase 2：锁定结局、核心真相和情绪落点，生成 `planning/ENDING-LOCK.md`
- Phase 3：从终点倒推章节/集功能，生成 `planning/REVERSE-OUTLINE.md`
- Phase 4：正文写作必须服务倒推大纲，不得为了单章爽点破坏终点
- Phase 5：审查时检查正文是否偏离倒推链

## 风险

- 结局未锁定时直接写正文，容易中途漂移
- 信息提前暴露，会削弱悬念或反转
- 误导线过强，会导致真实因果不可信
- 章节只追求局部精彩，会破坏整体弧线
