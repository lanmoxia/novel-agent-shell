# 连续性与项目记忆

## 核心原则

长篇和短篇都不应只依赖对话上下文维持一致性。项目稳定事实必须写入文件，供后续写作、审查、修订和锁稿读取。

## 必须记录的事实

- 人物身份、动机、已知信息和状态变化
- 时间线和空间关系
- 道具、线索、伏笔的状态
- 结局、核心真相、情绪落点
- 题材禁忌和项目专属禁止项
- 审查结论和修订影响

## 使用规则

- 写正文前读取 `PROJECT-MEMORY.md`
- 新增正文事实后更新 `PROJECT-MEMORY.md`
- 连续性审查必须以 `PROJECT-MEMORY.md` 为基准
- 修订或重写后必须记录事实变化
- 锁稿后不得改写既有稳定事实，只能追加锁稿和后置模块事实
