# 项目记忆更新协议

## 触发时机

- Phase 2 故事企划生成或确认后
- Phase 3 大纲和倒推规划生成或确认后
- Phase 4 每章/集正文生成后
- Phase 5 审查完成后
- Phase 6 修订或重写后
- Phase 7 锁稿后
- Phase 8 后置模块生成后

## 更新内容

| 阶段 | 必须更新 |
|------|----------|
| Phase 2 | 核心设定、结局、核心真相、人物初始设定、题材禁忌 |
| Phase 3 | 倒推链、章节功能、信息释放、伏笔回收 |
| Phase 4 | 新增事实、人物状态、道具状态、伏笔状态 |
| Phase 5 | 审查结论、风险点、建议处理方式 |
| Phase 6 | 修订原因、采用方案、影响文件、事实变化 |
| Phase 7 | 锁稿状态、最终正文路径、锁稿后限制 |
| Phase 8 | 后置输出类型、输出范围、不得反改正文 |

## 规则

- 素材、灵感和候选稿不自动成为稳定事实
- 只有用户确认或正式正文采用后，才写入稳定事实
- 不确定信息必须标注为“待确认”，不得伪装成已确认事实
- 记忆更新不得覆盖用户明确保留的旧设定
