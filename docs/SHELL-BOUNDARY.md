# 壳子边界说明

> 本文定义 novel-agent-shell-demo 作为“小说/剧本创作调度壳子”的边界。

当前壳子可以承载小说 / 剧本项目和教辅短剧项目的不同 Phase 语义，但边界不变：shell 负责流程，不负责具体题材、教学或视频生产规则。

---

## 1. 壳子负责什么

- 项目建档与阶段推进
- 题材 Skill 加载与锁定
- 项目目录结构与状态文件
- 审查派发、修订调度、锁稿管理
- 后置模块门禁
- 校验脚本与迁移脚本
- 运行记录与断点恢复
- 反馈与进化建议
- 按项目类型解释 Phase 0-8 的文档语义
- 记录 declared / inferred / missing capabilities，但不替 Skill 声明能力

---

## 2. 壳子不负责什么

- 不直接定义某个题材的写法
- 不直接塞入某个题材的文风、禁忌或案例
- 不把题材知识硬编码进主调度流程
- 不把项目偏好自动升级成全局规则
- 不自动改写锁定后的正式正文
- 不负责文言文解释正确性
- 不负责原文事实、白话释义或教学目标的业务判断
- 不负责 Scene 与原文映射的内容正确性
- 不负责字幕规则、分镜规则或视频生成风险的具体设计
- 不自动生成视频、图片或后置提示词
- 不绕过用户确认进入审查、修订、锁稿或后置输出

---

## 3. Skill 与知识库边界

### 3.1 全局写作知识库

面向所有题材，提供通用兜底能力，例如：

- 倒推设计
- 重写优先
- 连续性记忆
- 文风控制
- 协作注意事项
- 常见 AI 写作风险

当前合同见：

```text
docs/KNOWLEDGE-LAYERING.md
```

注意：全局写作知识库未来应作为 Skill 包或 Skill 资源承载，不在仓库根目录硬放具体知识库内容。

### 3.2 题材级知识库

面向单一题材，提供题材特有规则、案例、禁忌和审查标准。

教辅短剧 Skill 属于题材级知识库的扩展形态。它可以提供文言文原文拆解、白话释义、教学目标、人物 / 事件链、Scene 与原文映射、字幕、分镜、视频风险和教学审查规则，但仍不能接管 shell 流程。

### 3.3 项目级知识库

面向单个项目，记录该项目的稳定事实、规划、伏笔、信息释放和修订结果。

教辅短剧项目中，项目级知识库记录原文事实、句意解释、教学点、角色映射、Scene 映射、字幕规则、分镜风险、已确认产物和不可改写知识结论。

### 3.4 教辅短剧边界

教辅短剧项目的职责拆分：

| 层级 | 负责内容 | 不负责内容 |
|---|---|---|
| shell | Phase、状态、门禁、恢复、后置输出边界 | 具体释义、教学规则、视频画面规则 |
| Skill | 原文拆解规则、释义规则、教学目标、Scene 映射、字幕、分镜、审查标准 | 创建项目、修改 workflow、绕过门禁 |
| 项目模板 | 项目级事实、状态、产物、锁定边界 | 通用教学规则全文 |
| 审查产物 | 原文准确性、释义准确性、Scene 映射、字幕、镜头可生产性 | 自动锁稿或自动修订 |

`projects/_template-edu-shortdrama/` 是教辅短剧模板候选，不影响旧 `projects/_template/`。使用该模板时，shell 仍只管理阶段和门禁；文言文解释、原文映射、字幕规则、分镜规则和视频风险由 Skill / 模板 / 审查共同承担。

---

## 4. 状态与校验原则

- 人可读状态继续放在 Markdown
- 脚本判断优先读取机器可读状态
- 历史回填必须显式标注
- 新结构不应伪造旧历史
- 教辅短剧项目的 `FINAL-SCRIPT.md` 只代表最终剧本，不代表完整生产稿包
- 后置提示词、图片或视频生成必须保持默认关闭，并等待用户明确确认

---

## 5. 迁移原则

- 老项目可以补齐新结构
- 补齐时要标明是历史回填
- 不生成假的旧快照
- 不篡改已锁稿正文
