# 知识库分层说明

> 本文定义壳子如何理解和调度不同层级的写作知识。本文只规定边界和加载原则，不研究具体题材内容。

---

## 1. 三层知识库

### 1.1 全局知识库

适用于所有题材和所有项目，作为兜底机制。

建议承载方式：

```text
skills/global-writing-core/
```

或后续按实际测试结果放入某个全局 Skill 包中。当前壳子不在根目录维护全局知识库内容。

职责：
- 提供通用写作协作规范
- 提供通用文风、对白、连续性、改稿、审查原则
- 在题材 Skill 缺失某项能力时提供基础兜底

限制：
- 不替代题材 Skill
- 不覆盖项目稳定事实
- 不写入具体项目目录

### 1.2 题材知识库

适用于单个题材或类型。

当前位置：

```text
genre-skills/<skill-name>/
```

职责：
- 提供题材规则、文风、结构、人物、对白、禁忌、审查标准
- 提供模板、示例和参考资料

限制：
- 不控制项目流程
- 不创建或修改项目文件
- 不绕过壳子门禁

### 1.3 项目知识库

适用于单个创作项目。

当前位置：

```text
projects/<project-name>/PROJECT-MEMORY.md
projects/<project-name>/planning/
projects/<project-name>/materials/
```

职责：
- 记录项目稳定事实
- 记录结局、倒推链、伏笔、信息释放
- 记录用户项目偏好和修订结果

限制：
- 不自动升级为全局规则
- 不自动反向修改题材 Skill

---

## 2. 加载优先级

正式创作时优先级如下：

1. 项目知识库：当前项目已经确认的稳定事实优先级最高
2. 题材知识库：题材 Skill 提供写作规则和审查标准
3. 全局知识库：仅在通用协作、通用质量或缺失能力兜底时读取

冲突处理：

- 项目事实 > 题材规则 > 全局原则
- 题材禁忌 > 全局建议
- 用户明确确认的项目设定 > 示例内容

---

## 3. 兜底规则

如果题材 Skill 缺少某项能力：

- 缺少 style-guide：可在未来加载全局写作 Skill 的文风兜底；当前仅记录降级
- 缺少 dialogue-rules：可在未来加载全局写作 Skill 的对白兜底；当前仅记录降级
- 缺少 review-rubric：使用 `workflow/execution/review-rubric.md` 的流程评分兜底
- 缺少 taboo：只执行通用安全、逻辑和连续性检查，不编造题材禁忌
- 缺少 examples：不阻断，按规则文件生成

兜底必须在 `GENRE-SKILL-LOCK.md` 中记录。

---

## 4. 进化边界

反馈进入进化系统时，应先判断属于哪一层：

- 壳子流程问题 → workflow 或 scripts
- 全局写作问题 → 未来的全局写作 Skill 包
- 题材问题 → genre-skills/<skill-name>
- 项目偏好 → projects/<project-name>

不得把项目偏好直接升级为全局规则。

进化提案统一写入：

```text
workflow/feedback/proposals/
```

提案只说明建议位置和影响范围，必须等待用户确认后才修改对应层级。
