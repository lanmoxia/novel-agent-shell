# WENYAN-GENERALIZE-001-PLAN：第二篇文言文泛化测试计划

- 任务编号：WENYAN-GENERALIZE-001-PLAN
- 任务名称：规划《司马光》作为第二篇文言文泛化测试
- 完成日期：2026-05-29
- 状态：已完成计划制定，未进入正式创作执行

## 1. 任务背景

当前 `wenyan-skill` 已通过《两小儿辩日》完成第一轮文言文教辅真人古风短剧剧本创作链路验证，并在 `WENYAN-SMOKE-010-SCRIPT-CREATION-BOUNDARY-FREEZE` 中冻结边界。

本轮任务不是继续视频生产，也不是直接写第二篇剧本，而是规划第二篇文言文泛化测试。测试文本暂定为《司马光》，测试类型为事件危机型文言文教辅真人古风短剧泛化测试。

## 2. 当前已完成基线

当前已完成基线包括：

- 《两小儿辩日》烟测项目已完成 Phase 8 剧本文本锁稿。
- 锁稿项目为 `projects/wenyan-smoke-liangxiaor-bianri-001/`。
- 锁稿资产包括 `reviews/final-review.md`、`locked/FINAL-SCRIPT.md`、`locked/LOCK-MANIFEST.sha256` 和 `locked/SKILL-LOCK-MANIFEST.sha256`。
- `PROJECT-STATE.json` 已记录 `phase: 8`、`locked: true`、`productionPackageStatus: not_generated`。
- `WENYAN-SMOKE-010-SCRIPT-CREATION-BOUNDARY-FREEZE` 已明确当前主线不进入分镜提示词、Seedance2.0 提示词或视频生产链路。

该基线只证明《两小儿辩日》这一争辩型 / 观点证据型文本的剧本创作链路已跑通，不等于 `wenyan-skill` 已完成跨文本类型泛化验证。

## 3. 为什么需要第二篇泛化测试

第二篇泛化测试有必要，原因如下：

1. 第一篇《两小儿辩日》主要验证的是争辩型文本：人物提出相反观点、分别举证、结尾保留判断困难和原文反问。
2. `wenyan-skill` 的适用范围声明覆盖古代人物故事、古代寓言和有明确事件链的文本，不能只用一篇争辩型文本证明通用能力。
3. 教辅短剧生产常见文本不止观点争辩，也包括危机处理、行动解困、机智判断、后果反转等结构。
4. 第二篇测试可以检查现有模板是否会把所有文言文都套成“观点 + 证据”结构，从而暴露结构泛化不足。
5. 第二篇测试可以继续验证项目建档、原文拆解、Scene 映射、阶段审查和生产边界是否适用于不同文本形态。

## 4. 为什么选择《司马光》

《司马光》适合作为第二篇泛化测试，原因如下：

1. 它是常见小学文言文 / 古代人物故事，符合 `wenyan-skill` 的适用内容。
2. `wenyan-skill` 的 `SKILL.md` 和默认模板已将《司马光》列为典型篇目。
3. 文本短小，人物和事件清楚，适合作为低风险泛化测试。
4. 它具有明确危机事件：孩童游戏、儿童坠瓮、众人惊慌、司马光判断并击瓮救人。
5. 它能测试 `wenyan-skill` 是否能从争辩型结构切换到事件危机型 / 行动解困型结构。
6. 它的视频化风险可控，但仍能暴露儿童角色稳定性、危机表现尺度、动作复杂度和教学解释负载等问题。

本轮仅暂定测试文本为《司马光》，不在计划阶段确认具体教材版本、原文版本或正式剧本文本。

## 5. 《两小儿辩日》与《司马光》的结构差异

《两小儿辩日》属于争辩型 / 观点证据型：

- 入古境。
- 见事件。
- 起争辩。
- 摆观点。
- 举证据。
- 难判断。
- 回原文。
- 明主题。

《司马光》属于事件危机型 / 行动解困型：

- 入古境。
- 见孩童游戏。
- 突发坠瓮危机。
- 众人惊慌。
- 司马光判断。
- 击瓮救人。
- 回扣原文。
- 明方法与品质。

核心差异：

- 《两小儿辩日》的张力来自观点对立和证据对照。
- 《司马光》的张力来自突发危机、群体反应和关键人物行动。
- 《两小儿辩日》的教学重点是观点 / 证据、观察角度、判断边界。
- 《司马光》的教学重点是事件顺序、关键动作、沉着机智、解决问题的方法。
- 《两小儿辩日》不应加入现代科学答案；《司马光》不应夸大危险画面、动作戏或英雄化特效。

## 6. 本次泛化测试要验证什么

本次泛化测试要验证 `wenyan-skill` 的以下能力：

1. 能否先拆原文，再规划剧本，不直接跳到创作正文。
2. 能否识别《司马光》的事件链：游戏、坠瓮、众惊、司马光行动、儿童得救。
3. 能否把争辩型结构切换为事件危机型 / 行动解困型结构。
4. 能否让每个阶段性产物绑定原文句子、白话释义和教学功能。
5. 能否在儿童危机题材中保持教辅表达，不走惊悚、打斗、夸张灾难或短视频爽点。
6. 能否控制角色数量、场景数量和动作复杂度，保持真人古风短剧可拍性。
7. 能否保留生产边界：只验证剧本创作链路，不自动生成正式字幕、正式分镜、Seedance prompt 或生产稿包。
8. 能否继续沿用教辅项目模板和真实项目校验语义，不补旧小说规划文件。

## 7. 本次泛化测试不验证什么

本次泛化测试不验证：

- 分镜提示词能力。
- Seedance2.0 提示词能力。
- 视频提示词能力。
- 图像提示词能力。
- 正式字幕生产能力。
- 完整生产稿包能力。
- Phase 8 锁稿能力。
- 已锁稿《两小儿辩日》资产的可复用性。
- `VIDEO-SKILL-001-PLAN` 或任何视频后置 Skill。
- `wenyan-skill` 在所有文言文类型上的完整泛化能力。

本次测试最多验证《司马光》这一事件危机型文本的剧本创作链路可规划性和后续 RUN 边界。

## 13. 风险与边界

主要风险：

1. 把《司马光》误套成《两小儿辩日》的观点 / 证据结构，导致事件危机型文本失真。
2. 过度 dramatize 坠瓮危机，把教辅短剧写成惊险动作戏。
3. 过度英雄化司马光，削弱原文中的沉着、机智和方法意识。
4. 儿童角色、群体惊慌和击瓮动作在视频后置阶段存在稳定性风险，但本轮不验证视频生产。
5. 原文版本、教材版本和目标年级若不明确，会影响白话释义、教学目标和字幕负载。

边界：

- 本轮只完成计划文档和 AI-CONTEXT 状态同步。
- 本轮不创建《司马光》项目目录。
- 本轮不生成剧本、完整分场、完整台词或任何视频后置产物。
- 本轮不修改 `wenyan-skill`、workflow、scripts、模板或已锁稿项目资产。
